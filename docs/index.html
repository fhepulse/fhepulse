How It Works

Four steps to private governance

From poll creation to verifiable results—privacy guaranteed at every step by Fully Homomorphic Encryption.

01

Create a Poll

Define your options, choose linear or quadratic voting, set a credit budget, and deploy a dedicated smart contract.

02

Vote Privately

Responses are encrypted client-side before submission. Individual votes are never visible on-chain—not even to validators.

03

Tally On-Chain

The Fhenix network computes over encrypted ballots using FHE. Tallies are aggregated without decrypting any single vote.

04

Reveal Results

The poll creator triggers a multi-transaction decryption flow. Only the final aggregated totals are ever revealed.

Features

Built for private decision-making

Every design choice serves one goal: enabling collective decisions without compromising individual privacy.

FHE-Encrypted Voting

Votes are encrypted before hitting the chain. The network computes over ciphertext—no trusted third party, no reveals, no compromises.

Per-Poll Isolation

Each poll deploys its own smart contract via PollFactory. FHE permissions and state are fully isolated between polls.

Quadratic Voting

Support for quadratic voting mode where cost grows with conviction. Voters distribute credits across options—weight² against a budget.

Linear Voting

Classic weighted voting where you distribute points freely across options. Simple, transparent allocation for straightforward decisions.

Fully Decentralized

No backend needed for core voting. Polls live entirely on-chain. The API server provides indexing and UX enhancements, not trust assumptions.

Multi-Tx Finalization

Results are decrypted via a two-step process: requestFinalize() triggers FHE decryption, finalize() retrieves plaintext tallies.

Encrypted Inputs

Voters submit InEuint32[] arrays—one encrypted weight per option. These are converted on-chain via FHE.asEuint32() and accumulated into encrypted tallies.

FHE Access Control

Every encrypted value requires explicit permission grants: FHE.allowThis() for storage, FHE.allowSender() for returns, and FHE.allow() for specific addresses.

Conditional Logic

Branching on encrypted booleans is impossible. All conditional logic uses FHE.select(condition, ifTrue, ifFalse) to maintain the encryption invariant.

Two-Phase Decryption

Decryption is asynchronous. requestFinalize() submits a decryption request to the FHE co-processor. After processing, finalize() reads results via FHE.getDecryptResultSafe().
